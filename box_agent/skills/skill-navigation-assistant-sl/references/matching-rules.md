# 任务匹配规则

## 匹配目标

根据用户任务，在本地技能库中推荐最合适的前 3 个技能，并说明推荐理由、适用方式和组合路径。

## 推荐评分

- 描述语义匹配：35%
- 触发词匹配：20%
- 输入输出类型匹配：15%
- 任务场景匹配：15%
- 技能健康状态：10%
- 用户偏好或近期上下文：5%

## 匹配等级

- 高：可直接解决用户主要任务，或明显是主技能。
- 中：可解决任务中的一个关键环节，适合作为组合技能。
- 低：只有弱相关，不应强推。

## 组合判断

当任务包含多个阶段，例如调研、设计、生成、校验、打包、发布，应优先给出组合使用建议。组合建议必须说明执行顺序。

## 无合适技能判断

如果最高匹配度仍无法覆盖主要任务，或推荐理由只能停留在泛泛相关，应明确说明本地暂无足够合适技能，并进入 SkillHub 补齐建议。
