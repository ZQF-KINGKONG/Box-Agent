# SkillHub 补齐策略

## 触发条件

- 本地技能库没有高匹配技能。
- 用户明确要求去 SkillHub 搜索。
- 本地技能存在但不可调用或健康检查异常。
- 用户任务需要明显缺失的新能力。

## 推荐原则

优先推荐：

1. 与任务目标直接匹配的技能。
2. 有清晰说明、版本信息和根目录 SKILL.md 的技能。
3. 近期维护、来源可信、风险较低的技能。
4. 可安装、可校验、可启动的技能。

## SkillHub 安装入口

根据 SkillHub 技能详情页的常见呈现，技能安装通常可以有两种入口：

1. **Prompt 安装**：页面提供“将提示词发送给你的 AI 安装该 skills / 复制 prompt”一类入口。此方式适合把安装指令交给当前 AI 助手继续执行。
2. **ZIP 包安装**：页面提供技能包下载入口时，可下载 ZIP 后进行本地结构校验和安装。

因此，本技能不应把 SkillHub 安装理解为必须依赖单一协议；应优先识别页面上的可用入口，并向用户说明推荐方式。

## 安装确认

无论使用 Prompt 安装还是 ZIP 包安装，只要涉及下载、安装、覆盖、执行安装脚本、移动文件、启动需要额外权限的技能，都必须获得用户明确确认。

## 安装前校验

若使用 ZIP 包安装，安装前应检查：

- ZIP 根目录是否有包级 `SKILL.md`。
- YAML front matter 是否存在且可解析。
- name、description、version 等关键字段是否完整。
- 是否存在可疑脚本或高风险权限要求。
- 是否与本地已有技能重名。
