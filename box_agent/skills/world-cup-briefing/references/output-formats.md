# 输出格式

## 默认语言与风格

- 默认使用中文。
- 默认风格为“球迷向”：有比赛情绪、有故事线、有看点，但不夸大事实。
- 适合快速阅读，优先使用小标题和要点。
- 可以有热血表达，但不得把预测、传闻写成事实。

## 世界杯战况简报 HTML 格式

用于“生成世界杯简报”“昨天世界杯战况”“最近世界杯有什么比赛”等请求。

硬性范围：

- 默认生成“昨日世界杯综合战况简报包”，不得只覆盖单场比赛，除非用户明确指定某一场。
- 如果用户要求“最近”“这几天”“近况”，最多只覆盖最近 3 天；仍应按综合简报组织多场信息。
- 不写世界杯历史、赛事泛介绍、年度特色、城市文化、花边软文等无关内容。
- 只保留赛果、关键事件、积分/出线影响、关键人员、用户关注对象等与战况直接相关的信息。

默认交付：

- 如果用户没有提出其他明确格式要求，必须输出一个美观的单页 HTML 文件；聊天中可附 3-5 条摘要。
- 如果用户明确要求图片、海报、长图、社媒卡片，可以生成图片版；图片版应保留同样的信息范围、关键比分/赛程和关键人员信息。
- HTML 需要适合直接保存为 `.html` 文件或在浏览器中打开。
- 视觉风格应有球迷向氛围：强对比色、比分卡片、球队标签、关键人物卡片、醒目的焦点战区块。
- 页面必须包含数据更新时间和信息范围，例如“昨日战况”或“最近 3 天战况”。

HTML 页面建议结构：

1. **Hero 区**  
   包含标题、信息范围、更新时间、一句话战况情绪摘要。

2. **战况总览卡片**  
   展示比赛数量、总进球、爆冷/强强对话、出线或排名影响等可核验指标；没有数据时不要编造。若范围内有多场比赛，必须体现多场覆盖。

3. **已完赛战报 / 比赛结果区**  
   按比赛列出对阵、比分、阶段、进球/红黄牌/点球等关键事件；默认覆盖范围内所有已完赛重点比赛，不得抓一场就收工。

4. **焦点战复盘**  
   只选最重要的 1-3 场，写比赛转折、节奏、胜负手。

5. **关键人员信息**  
   必须出现。覆盖核心球员、进球/助攻者、门将/防线核心、主教练、伤停/停赛、替补奇兵、状态变化或战术角色。

6. **用户关注重点**  
   如果用户关注特定球队/球员，例如巴西、内马尔，应单独做卡片；如果最近 3 天没有相关比赛或动态，需要明确写“本期未检到直接相关战况”，不要硬凑。

7. **今日/明日赛程预告与下一步关注**  
   只写未来 24 小时左右的重点赛程、人员状态和直接相关看点 2-4 条，不做泛泛介绍。

8. **信息来源与待确认**  
   标明已确认结果、更新时间、来源类型；对伤停/首发/赛程冲突等写明“待确认”或“未检到官方确认”。

## 焦点战预告格式

用于“焦点战预告”“今晚哪场最值得看”“写某场比赛前瞻”等请求。

建议结构：

1. **比赛信息**  
   对阵双方、时间、阶段、场地、出线/淘汰背景。时间敏感信息必须核对后再写。

2. **为什么这场值得看**  
   用球迷视角提炼比赛张力，例如宿敌、巨星对话、风格冲突、复仇线、出线生死战。

3. **关键人员信息**  
   必须出现。分别写双方核心球员、关键替补、主教练、伤停/停赛、可能首发与关键对位。

4. **战术对位**  
   比较阵型、压迫、反击、边路、定位球、中场控制、防线弱点等。

5. **胜负手**  
   列出 2-4 个可能决定比赛走势的因素。

6. **用户关注视角**  
   如果比赛涉及用户喜欢的球队/球员，应加强分析；如果不直接相关，可补一句他们与本场结果的间接关系。

7. **大胆但克制的判断**  
   可以给趋势判断，但要明确预测不是事实。

## 球员/球队更新格式

用于“更新内马尔情况”“巴西现在怎么样”等请求。

1. **当前状态**
2. **关键人员信息**
3. **对球队/比赛的影响**
4. **球迷视角看点**
5. **后续关注**
