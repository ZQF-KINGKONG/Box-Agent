# 质量检查清单

在输出世界杯简报、焦点战预告、球队/球员更新前，逐项检查：

- [ ] 是否识别清楚用户要的是简报、焦点战预告、球队更新、球员更新，还是偏好更新？
- [ ] 用户未指定单场/球队/球员时，是否生成“昨日/当日综合简报包”，而不是单场战报？
- [ ] 如果范围内存在多场比赛，是否覆盖多场已完赛重点比赛？
- [ ] 是否包含今日/明日或未来 24 小时重点赛程预告？
- [ ] 如果用户没有提出其他明确格式要求，是否真的生成了可打开的 `.html` 文件？
- [ ] 如果用户明确要求图片/海报/长图/社媒卡片，是否生成了可读、可分享且保留关键人员信息的图片版？
- [ ] 是否给出来源、更新时间、已确认项与待确认项说明？
- [ ] 是否使用中文输出？
- [ ] 如果是简报，是否默认只覆盖昨天战况，或在用户要求最近近况时最多覆盖最近 3 天？
- [ ] 如果是简报，是否避免了世界杯历史、赛事泛介绍、年度特色、城市文化、花边软文等无关内容？
- [ ] 如果是简报，是否默认输出美观的单页 HTML？
- [ ] 是否符合默认“球迷向”风格，同时保持事实克制？
- [ ] 是否包含清晰的“关键人员信息”小节或等价内容？
- [ ] 关键人员是否按比赛影响自动筛选，而不是只写用户举例点名的人？
- [ ] 是否覆盖进球/助攻/扑救/红黄牌/关键失误/战术变化/伤停等直接影响比赛的人？
- [ ] 是否覆盖双方/相关球队的核心球员与主教练？
- [ ] 是否说明伤停、停赛、阵容、首发、赛程、比分等时间敏感信息的来源或核验状态？
- [ ] 是否区分已确认事实、传闻、不确定信息和个人判断？
- [ ] 是否体现用户偏好，例如巴西、内马尔，但没有牺牲整体完整性？
- [ ] 是否有球迷感的看点、情绪和比赛故事线？
- [ ] 是否避免编造比分、名单、伤病、采访或转会传闻？

## 最低合格标准

如果是简报或焦点战预告，至少必须包含：

1. 对简报而言，是昨日/当日综合范围；对焦点战而言，是明确比赛背景。
2. 关键人员信息。
3. 主要看点或胜负手。
4. 用户关注对象相关信息，若适用。
5. 对不确定信息的明确标注。
