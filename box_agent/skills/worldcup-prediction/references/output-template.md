# 预测报告输出模板

LLM 在 `predict.py` 的 JSON 基线上,按本模板写**中文单场**预测报告。所有数字(xG、胜平负、
置信度、比分)必须取自脚本输出,不得自行编造;定性叙述(对位、进球剧本)基于数据推演。
**只做单场**,不出「未来 24 小时」批量表。

---

## 单场模板

```
# {主队名} vs {客队名}  预测

## 预测结果
- 最可能结果:{result_label}({result_prob}%) → 预测比分 {predicted_score}
- 置信度:{tier}({percent}%)  · xG:{主队} {xg_a} - {xg_b} {客队}
- 胜平负:{主队}胜 {win}% | 平 {draw}% | {客队}胜 {loss}%
- 最可能比分:{score1} {p1}% / {score2} {p2}% / {score3} {p3}%
  (注:单一最高比分 {modal_scoreline} 多为平局,头条以「最可能结果」为准)

## 三线与阵容(必写)
读 JSON `comparison.lines` 与 `comparison.squad`:
- 三线:前场 {主}{atk_a}-{atk_b}{客}、中场 {mid}、后场 {def}、速度 {pace} —— 各条谁占优、差多少
- 阵容:核心平均年龄 {主}{age_a}-{age_b}{客} 岁、年龄结构 {age_balance}、经验 {experience}、
  状态 {form} —— 谁更老练/更冲、状态谁更热

## 对位拆解(高精度,重点)
读 JSON `matchup` + `xg_breakdown`,逐方写「基线 ×动机 ×对位 = 最终 xG」,并逐维点评:
- {主队}进攻:基线{base}→最终{final}(对位×{m_mult}) —— 边路速度 {wing对位+效应}、
  一对一 {dribble对位}、反击打身后 {counter对位}、空中/定位球 {aerial对位};指出最吃这条线的弱点
- {客队}进攻:同上 —— 哪条线能打、哪条被掐死
- 一句话:对位整体谁占上风,比分因此被抬高/压低多少

## 防守端(必写)
读 JSON `defense`,逐方写本方防线状况:
- {主队}防线:被对手对位×{concede_xg_mult};最吃【{weaknesses[0].dim}】({+x%}) —— {read};
  还吃哪几条({其余 weaknesses});哪条守得住({solid})
- {客队}防线:同上
- 结论:谁的防线是这场的隐患/胜负手(结合上面对位拆解)

## 关键分析
1. 后防失误概率:{主队} {a_err}% vs {客队} {b_err}% —— {谁的防线更易出错,是否会送失误球}
2. 中场强度:{主队} {a_mf} vs {客队} {b_mf}(差 {diff}) —— {谁掌控节奏、压制或反制}
3. 经验与年龄平衡:{对比 experience/youth/age_balance} —— {大赛沉稳还是冲劲足,谁更稳}
4. 核心站位对防线的影响:{挑出对方最危险的核心球员 position+traits} —— {压哪一侧、
   对本方哪个防守者/区域造成威胁}
5. 对位结论:{核心威胁点 vs 本方对应防守区域} —— {是否吃亏、谁占优、决定比赛走向}

## 赛况与策略(必填,体现"算得深")
- 积分/出线形势:{本组现状,各队积分+净胜球,谁主动谁被动}
- 净胜球:{首轮平/小胜时,本场净胜球对排名/出线的意义,是否需要刷大比分}
- 争第一还是第二:{该队想拿小组头名还是次名?对应淘汰赛对阵/半区,是否值得全力}
- 伤病复出时间线:{关键伤员本场能否上?预计第几轮/淘汰赛回归?已反映在阵容/评分}
- 动机与轮换:{是否必胜/可轮换/死亡之组/无关痛痒?如已用 xg_mult 体现,说明乘子与理由}

## 进球剧本
- {主队}进球:{xG份额最高者姓名}({位置}) —— {从 scoring_styles 选具体方式,
   结合对方后防失误/对位弱点,写怎么进的}
- {客队}进球:{xG份额最高者姓名}({位置}) —— {同上}

## 一句话总结
{综合上面的对位与失误分析,给出比赛走向与风险点}

## 想要对决图?(此节必带,不可省)
> 需要的话,我可以生成一张「{主队} vs {客队}」世界杯对决图,包含预测比分、胜平负概率、
> 关键球员与进球方式,适合发群或做赛前海报。
> 本场图方向:**{predicted_score}｜{主队双星} vs {客队双星}｜{主队色}×{客队色}对抗风｜{stage}预测卡**
> 回复一声,我就出图(`scripts/card.py` 生成生图 prompt → 客户端生图模型渲染)。
```

> 上面 ``` 内即完整报告,**最后一节「想要对决图?」必须保留**——这是默认收尾,不能省。
> {主队双星}/{客队双星} = 各队 `likely_scorers` 前两名;{队色}由球队主色描述(如蓝金/红绿)。

## 规则

- **头条跟随胜平负**:`predicted_score` 已是「最可能结果对应的最可能比分」,直接用它当头条;
  不要把 `modal_scoreline`(常 1-1)当结论。中性场不提主场加成。
- **声明实时校正**:若用了 `--overrides`,报告开头必须用一句话说明对哪些队做了什么校正
  (取自 JSON `adjustments` 的 `note` + `changes`),例:「荷兰已按后防三主力伤停下调
  defense -7」。没用实时则注明「基于整体基准画像,未叠加实时」。
- **数字必抄脚本**:比分、xG、胜平负%、置信度%、top 比分,全部来自 JSON,不四舍五入
  之外的加工。
- **定性必落到数据**:每条分析引用具体球员/评分,不空泛。例:不只说「中场强」,要说
  「罗德里 + 佩德里掌控节奏,把对方中场压制 6 分」。
- **进球剧本要具体**:谁进、用什么方式(scoring_styles)、为什么能进(对位吃亏/失误/
  反击),写清楚。
- **置信度低要解释**:当置信度为「低」时,说明原因(势均力敌/数据接近/高方差)。
- **不编造未提供的数据**:球员伤停、临场状态若无输入,不要捏造;可提示「假设全员健康」。
