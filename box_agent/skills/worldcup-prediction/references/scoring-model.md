# 评分模型说明

本文档解释 `scripts/predict.py` 的评分逻辑。当 Python 不可用时,LLM 可按此手工推算
(作为降级方案);预测报告的叙述也应以此模型为依据,保证可解释、可复现。

## 数据说明(重要)

- `references/teams/*.yaml` 中的评分(ratings 0-100)与球员属性是**主观近似估计**,
  非权威统计。用于玩具级预测,不代表官方数据。已知真实伤停/状态变化时应手动更新。
- 球员姓名、位置、大致风格为公开常识;具体数值为估计。

## 1. 期望进球 xG

对「进攻方 vs 防守方」计算:

```
base            = 1.35                       # 联赛平均基线
ratio           = attack / defense            # 攻防比
midfield_factor = 1 + (midfield_攻 - midfield_守)/100 * 0.35   # 中场控制(重点)
form_factor     = 0.85 + (form - 70)/100      # 近期状态
error_bonus     = (defensive_error_prob_守 / 100) * 0.6        # 后防失误加成(重点)

xG = base * ratio * midfield_factor * form_factor + error_bonus
若为主场: xG *= 1.10
下限钳制: xG >= 0.2
```

- **中场强度**(重点):差值越大,xG 偏移越多(±0.35 区间)。
- **后防失误概率**(重点):对方失误概率越高,本方 xG 越高(最多 +0.6)。
- 双方各算一次:xG_a = f(A攻, B守),xG_b = f(B攻, A守)。

## 2. 比分概率(泊松分布)

对每个进球数 a,b ∈ [0,6]:

```
P(a,b) = Poisson(a; xG_a) * Poisson(b; xG_b)
Poisson(k; λ) = e^(-λ) * λ^k / k!
```

- 胜/平/负 = 把所有比分按 a>b / a==b / a<b 聚加(以 team_a 视角)。
- **头条比分 `predicted_score`(重点修正)**:独立泊松在两队 xG 都落在约 1.2–1.8 时,
  *联合众数*几乎必然是 1-1,会和「明显占优一方」矛盾。因此分两步:
  1. 先取**最可能结果** `most_likely_result` = max(胜, 平, 负);
  2. 再在该结果对应的比分里(胜→a>b / 平→a==b / 负→a<b)取概率最高者作为 `predicted_score`。
  - 同时保留 `modal_scoreline`(原始联合众数,常为 1-1)供透明对照,但**不作头条**。
  - 例:阿根廷 xG 1.6 vs 克罗地亚 1.24 → 胜平负 46/25/30,头条取「阿根廷胜」下最高的 1-0,
    而非联合众数 1-1。

### 中性场 vs 主场

- 世界杯小组赛多为中性场:`schedule.yaml` 标 `neutral: true`,xG **不**乘 1.10。
- 仅东道主(美/加/墨)在本国比赛 `neutral: false`,主队 xG ×1.10。
- 指定单场时,默认中性;加 `--home <code>` 才给该队主场加成。

### 高精度对位引擎 matchup_engine(后防与对位直接修正 xG)

基线 xG 只看整体攻防/中场;真正决定单场的是**具体对位**。引擎从已有球员/球队数据派生 4 条
对位线,各产生一个有界的 xG 效应,乘成该方的"对位乘子",**默认开启**:

| 对位维度 | 进攻方取值 | 防守方取值 | xG 效应区间 |
|----------|-----------|-----------|-------------|
| 边路速度冲击 | 最快边锋 pace | 对方整体 pace | -6% ~ +10% |
| 一对一/盘带 | 最强前场 dribbling | 对方 defense | -5% ~ +7% |
| 反击/打身后 | 反击/单刀型前锋 pace | 对方 pace + 失误率加成 | -4% ~ +8% |
| 空中/定位球 | 头球/定位球型射手 shooting | 对方 defense×0.6+experience×0.4 | -3% ~ +5% |

- 对位乘子 = clamp(1 + Σ各维效应, 0.82, 1.22)。
- 伤停/换人通过 `--overrides` 改 ratings/球员后,对位引擎自动用**校正后**的阵容重算。
- 中场强度不在对位乘子里(已在基线 `midfield_factor`),避免重复计;但仍在报告里点评。

**完整 xG 管线**:`xG = 基线(攻防比×中场×状态+失误) × 主场 × 对位乘子 × 动机乘子`,下限 0.2。
JSON 的 `xg_breakdown` 透明给出 base / matchup_mult / motivation_mult / final。

### 动机/策略乘子 xg_mult(算得深的关键)

比分不只取决于纸面实力,还取决于**这场球队想干嘛**。`--overrides` 里每队可给 `xg_mult`
(默认 1.0),在中性/主场加成之后再乘到该队 xG 上:

```
xG_最终 = clamp( xG_模型 * xg_mult , 下限 0.2 )
```

常见取值(由 web 查到的赛况推导,写进报告「赛况与策略」段):

| 情境 | xg_mult |
|------|---------|
| 必须取胜 + 刷净胜球争小组头名 | 1.05 ~ 1.15 |
| 正常全力 | 1.00 |
| 已基本出线、轮换保人/保平、为淘汰赛留力 | 0.85 ~ 0.95 |
| 濒临出局摆大巴、力求不被大比分 | 0.88 ~ 0.95(弱队进攻端) |

要点:伤员能否复出用 `drop_players`/`ratings` 表达;**净胜球诉求、争第一还是第二、轮换
与否**这类"意图"用 `xg_mult` 表达。两者叠加,才是"结合实时与赛况"的完整预测。

## 3. 置信度

```
concentration = 1 - entropy(比分分布) / max_entropy   # 分布集中度,0~1
top_result    = max(胜, 平, 负)                       # 结果决断度
confidence    = 35 + concentration*45 + (top_result - 0.33)*30
钳制到 [30, 95]
```

分级:

| 百分比 | 等级 |
|--------|------|
| ≥70    | 高   |
| 55-69  | 中   |
| <55    | 低   |

- 两强相遇、势均力敌 → 比分分布发散 → 置信度低(符合直觉)。
- 一方明显占优 → 分布集中、结果决断 → 置信度高。

## 4. 进球者与进球方式

按球员 `shooting` 属性在队内分配 xG 份额:

```
share_i = shooting_i / Σ shooting
expected_goals_share_i = xG_队 * share_i
```

- 排序输出最可能进球者,并附 `scoring_styles`(怎么进)与 `traits`。
- 叙述「谁进球、怎么进」时,优先从 xG 份额最高者取材,结合对方后防弱点。

## 5. 重点分析维度(叙述必填)

预测报告必须覆盖用户点名的重点,数据来源见 `comparison` 字段:

1. **后防失误概率**对比(`defensive_error_prob`)——谁的防线更易送失误球。
2. **中场强度**对比(`midfield` + `midfield_diff`)——谁掌控节奏、压制对方。
3. **经验与年龄平衡**(`experience` + `youth` + `age_balance`)——大赛沉稳 vs 冲劲。
4. **对面核心站位对防线的影响**——读核心球员 `position` + `traits`,判断对哪一侧/哪种
   防守者造成威胁(例:左脚内切型 RW 压对方左后卫)。
5. **对位是否吃亏**——把对方核心威胁点与本方对应防守区域的强弱对照,给结论。
