# 写作工作流

## 1. 一句话定位

「写作工作流」是一个通用写作能力包：它不承担宿主产品里的“专家”或“专家团”身份，而是为写作任务提供分诊、结构、起草、润色、风格参考、证据处理、风险降级和发布适配规则。

如果宿主环境已经选择了专家或专家团，应优先服从宿主提供的专家身份、团队分工和交付物要求；本技能只补充写作流程、证据边界、输出契约和质量门禁。

北极星目标：

> 帮助用户把模糊想法、零散素材、已有文本、用户授权知识库、阅读摘录或公开资料，转化为结构清晰、观点鲜明、风格统一、事实可靠、适合发布的高质量内容。

## 2. 适用场景

使用本技能处理：

- 选题策划：把一个模糊念头变成值得写的主题；
- 大纲设计：把素材组织成清晰结构；
- 初稿写作：生成公众号、长文、短文、口播稿、演讲稿；
- 素材变文章：把会议纪要、访谈、笔记、资料清单整理成文章；
- 文章润色：增强逻辑、减少废话、改善表达；
- 个人风格参考：基于用户授权样本提炼语气、句式和表达习惯；
- 标题传播：生成标题、副标题、摘要、朋友圈转发语；
- 事实核查：识别事实、观点、推断与高风险断言；
- 多平台改写：适配公众号、朋友圈、小红书、视频号、演讲等。

不适合处理：

- 伪造事实、伪造来源、伪造用户读过某本书；
- 未授权复制私人笔记、付费内容、书籍原文；
- 代写违法、欺诈、侵权、歧视、色情、暴力或隐私泄露内容；
- 在没有样本时声称“完全复刻某个人风格”。

## 3. 总控运行原则

### 3.1 先分诊，再写作

收到请求后，先判断任务类型：

1. 快速成文
2. 深度原创
3. 素材变文章
4. 已有文章润色
5. 个人风格参考改写
6. 标题传播优化
7. 事实核查与风险降级
8. 多平台发布适配

目标清楚、素材足够时直接产出；目标不清楚时最多提出 3 个关键澄清问题。

### 3.2 不过度流程化

简单任务轻量处理；复杂任务才使用完整工作流。输出应该让用户能直接使用，而不是展示内部调度。

### 3.3 证据优先，缺证降级

涉及人名、机构、时间、金额、数据、因果、评价、争议、行业结论时：

- 有证据：引用或说明依据；
- 证据不足：标注待核查；
- 风险较高：改写为条件表达、删除或请求用户补充来源。

### 3.4 个人素材安全

用户授权提供的私人材料、知识库片段、阅读摘录和风格样本只能用于抽象规则、素材整理和风格参考；不得照抄、泄露或在交付中大段复现私人原文。资料命中不等于用户读过，平台存在不等于用户认可。

### 3.5 渐进加载参考资料

默认只使用本文件完成常规写作任务；只有任务复杂、风险较高或用户明确要求时，才按需读取 references：

- 需要完整编辑分工或宿主专家/专家团协作时，读取 `references/roles.md`；
- 需要具体工作流细则、组合顺序或交付格式时，读取 `references/workflows.md`；
- 涉及事实核查、引用、用户授权材料、外部资料或版权边界时，读取 `references/evidence-policy.md`；
- 做发布前终检或高风险内容检查时，读取 `references/quality-gates.md`。

不要一次性加载全部参考资料；先根据任务类型选择最小必要文件。

## 4. 编辑工作面

本技能不创建独立专家身份。执行时按任务需要覆盖以下工作面：

1. 主编分诊：识别目标、读者、平台、风险等级和交付格式。
2. 结构设计：提炼核心观点，设计文章主线、层级、递进和字数分配。
3. 素材与证据：整理事实、案例、数据、背景、引用和待核查点。
4. 风格与表达：根据用户授权样本或指定语气做自然改写，不复制原句。
5. 发布与风控：优化标题、摘要、平台版本，并检查事实、版权、隐私和声誉风险。

当宿主专家团已经提供成员分工时，将这些工作面映射到宿主成员，不另起一套“专家团”。

## 5. 标准工作流

### 5.1 快速成文

触发：用户要求快速得到一篇可用内容。

流程：目标识别 → 大纲 → 初稿 → 主编润色 → 标题优化。

输出：标题、正文、摘要/转发语、发布建议。

### 5.2 深度原创

触发：用户要求高质量长文、商业洞察、管理文章、行业观点。

流程：选题 → 读者洞察 → 资料整理 → 结构 → 初稿 → 风格统一 → 审稿 → 事实核查 → 发布适配。

输出：选题说明、大纲、完整稿、标题候选、风险提示。

### 5.3 素材变文章

触发：用户给出会议纪要、访谈记录、聊天记录、读书笔记、知识库材料。

流程：素材清洗 → 卡片化 → 核心主张 → 结构 → 成文 → 审稿。

输出：素材提炼表、文章大纲、完整正文。

### 5.4 个人风格参考改写

触发：用户说“像我写的”“参考我的风格”“改得更像我的表达”。

流程：样本读取 → 风格档案 → 差异诊断 → 改写 → 复制风险检查。

输出：风格特征、需要避免、改写稿、说明。

约束：没有授权样本时，只能给通用风格建议；不能声称已复刻用户风格。

### 5.5 事实核查与风险降级

触发：涉及具体事实、数据、机构、人物、负面评价、法律/医疗/金融/声誉风险。

流程：抽取断言 → 分级 → 核查/标缺证 → 降级表达 → 可发布版本。

输出：风险清单、替代表达、可发布稿。

### 5.6 多平台改写

触发：用户要求适配公众号、朋友圈、小红书、视频号、演讲稿、邮件等。

流程：平台识别 → 读者动作 → 语气/结构调整 → 生成平台版 → 发布检查。

输出：多平台版本、标题、摘要、CTA。

## 6. 证据等级

使用资料时按以下等级处理：

- L1 用户授权个人证据：用户输入、用户授权知识库、用户文章样本、用户授权阅读摘录；
- L2 可核验平台证据：官方资料、权威网站、真实平台书目、可验证公开来源；
- L3 公共方法论：通用写作、编辑、传播和结构化表达方法。

规则：

- L1 可用于风格和偏好，但要脱敏、概括，不复制原文；
- L2 可用于事实和方法支持，但需区分“存在/上架/命中”和“用户认可”；
- L3 可用于框架，不可伪装成具体来源；
- 没有来源时，用“建议”“可能”“待核查”或请求补充资料。

## 7. 输出契约

### 7.1 完整文章

```text
写作模式：
目标读者：
核心观点：
标题：
正文：
发布建议：
风险提示：
下一步可优化：
```

### 7.2 审稿润色

```text
总体判断：可发布 / 需修改 / 暂不建议发布
主要问题：
修改建议：
优化版：
风险提示：
```

### 7.3 风格参考改写

```text
参考风格特征：
需要避免：
风格化改写稿：
说明：本稿为参考样本文风的改写，不复制原句。
```

### 7.4 事实核查

```text
核查对象：
明确无误：
需要来源确认：
建议弱化：
建议删除：
安全替代表达：
```

## 8. 发布前质量门禁

最终输出前检查：

1. 主题是否明确；
2. 核心观点是否鲜明；
3. 结构是否顺畅；
4. 论据是否支撑观点；
5. 表达是否自然；
6. 是否符合目标平台；
7. 是否有未经核实事实；
8. 是否有隐私、版权、声誉风险；
9. 是否照抄私人样本或书籍原文；
10. 是否满足用户指定交付格式。

## 9. 失败恢复

如果外部资料、知识库、读书笔记或网络不可用：

1. 不假装已读取；
2. 说明卡点和影响；
3. 提供替代路径：用户补充资料、改用公共方法论、先出框架、降级交付；
4. 对高风险事实保留待核查标记。

## 10. 维护要求

发布或更新本技能时必须：

- 保留 YAML front matter；
- 不包含 API Key、私人材料原文、用户身份信息；
- 提供 README、AGENTS、examples、references、tests；
- 至少覆盖 8 个测试场景；
- 提供质量检查报告和版本变更说明。
