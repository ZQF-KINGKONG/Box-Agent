# 写作工作流 Skill 包：证据等级与外部资料入口参考文档

## 1. 目标与适用范围

本文档用于规范「写作工作流」Skill 在执行写作、改写、观点提炼、书单推荐、方法论整理、案例分析等任务时，如何识别、调用、验证与引用不同类型的证据来源。

核心目标是：

- 优先使用最贴近用户本人经验、偏好与真实业务场景的证据；
- 区分“可作为写作启发的资料”与“已验证、可直接支撑结论的证据”；
- 避免把搜索命中、平台存在、模型记忆或公共方法论误当作事实核验；
- 在保护用户隐私与版权边界的前提下，合理利用用户授权材料、个人知识库、阅读摘录和网络权威资料。

适用对象包括但不限于：

- 用户授权知识库、项目材料或笔记系统；
- 用户授权阅读摘录、书目信息、读书笔记或评论；
- 网络权威资料、官方页面、论文、报告、标准、机构发布信息；
- 公共写作方法论、行业通用框架、公开课程或书籍中的一般性观点。

---

## 2. 证据等级原则

「写作工作流」Skill 应按照以下优先级使用证据：

> 用户个人证据 > 真实平台证据 > 公共方法论证据

### 2.1 L1：用户个人证据（最高优先级）

用户个人证据是指直接来自用户本人、用户授权知识库、用户历史表达、个人笔记、授权阅读摘录、会议纪要、业务文档、项目材料等内容的证据。

典型来源：

- 用户直接输入的观点、要求、经历、案例；
- 用户授权知识库、项目材料或笔记系统中的内容；
- 用户上传的文件、会议纪要、复盘、方案、周报；
- 用户授权阅读摘录、个人笔记、书评或评论；
- 用户在当前会话或历史记忆中明确表达过的偏好、判断标准、风格要求。

使用原则：

- 优先作为写作方向、论点、风格、案例和判断标准的依据；
- 当用户个人证据与公共方法论冲突时，默认以用户个人证据为准，除非用户明确要求采用外部框架；
- 用户授权阅读摘录、个人笔记、个人评论的优先级高于单纯的“书籍已上架”或“搜索命中”；
- 可提炼、概括、改写用户个人证据中的观点，但不得未经允许大段复制私人原文；
- 涉及敏感、隐私、商业机密内容时，应做必要脱敏，避免在最终交付中暴露不必要的个人或业务细节。

### 2.2 L2：真实平台证据

真实平台证据是指来自可信平台、官方渠道或可核验入口的事实性资料。其价值高于通用方法论，但低于用户个人证据。

典型来源：

- 公开或授权平台中实际可检索到的书目、作者、出版社、简介、目录等；
- 官方网站、政府机构、行业协会、标准组织发布的信息；
- 学术数据库、论文页面、出版社页面、企业官方公告；
- 权威媒体、研究机构、咨询公司发布的报告；
- 平台内可确认的阅读状态、收藏状态、划线状态或用户授权数据。

使用原则：

- 平台“存在”只能证明资料入口存在，不自动证明其中观点适合当前写作；
- 搜索命中不等于已验证。搜索结果只代表可能相关，需要进一步打开、核对来源、时间、作者、上下文和可用性；
- 书目命中不等于用户读过、认可或做过摘录；
- 若能获取用户授权阅读摘录或笔记，应优先使用摘录和笔记，而不是只引用书籍简介；
- 引用网络资料时，应尽量使用官方、原始、近源资料，而非二手转述；
- 对数据、年份、定义、政策、标准等事实性内容，必须核对来源和发布时间，避免使用过期信息。

### 2.3 L3：公共方法论证据

公共方法论证据是指广泛流传、可作为写作辅助框架的一般性知识、模型、方法、理论或常识性经验。

典型来源：

- 金字塔原理、SCQA、AIDA、STAR、MECE、5W1H 等通用表达框架；
- 公开书籍、课程、文章中的一般性写作方法；
- 行业中常见的管理、营销、沟通、复盘、用户研究方法；
- 模型内置的通用知识与常见表达经验。

使用原则：

- 主要用于搭建结构、改进表达、补全论证链路；
- 不应替代用户真实经历、业务事实或平台可核验证据；
- 不应伪装成用户个人洞察或用户真实案例；
- 当公共方法论与用户指定风格冲突时，应服从用户风格；
- 若方法论来源需要严肃引用，应补充真实来源核验，不应仅凭模型记忆断言出处。

## 3. 外部资料入口规范

### 3.1 授权知识库 / 笔记入口

使用目的：

- 查找用户已有观点、长期偏好、项目背景、业务案例、过往表达方式；
- 提取用户自己认可过的概念、判断、案例和话术；
- 保持新内容与用户知识体系一致。

使用要求：

- 优先检索与当前主题高度相关的笔记、项目文档、历史材料；
- 对检索到的内容进行摘要和观点提炼，而不是机械复制；
- 对不确定的上下文做保守处理，不把片段误读为完整立场；
- 若内容属于私人记录，最终交付中应避免暴露原始记录细节，除非用户明确要求引用。

### 3.2 授权阅读材料 / 书目信息入口

使用目的：

- 核验用户关注的书籍是否存在、版本与书名作者是否准确；
- 参考用户授权阅读摘录和笔记，判断用户真正关注的章节、概念和观点；
- 在推荐书单、提炼读书观点、写作素材整理时提高个性化程度。

使用要求：

- 推荐书籍时，应优先核验书籍是否真实存在、版本是否可确认；
- 书目搜索命中只说明平台存在该书，不代表用户已阅读或认可；
- 用户授权阅读摘录、笔记、书评优先级高于书籍简介、目录和平台推荐语；
- 不得复制用户私人摘录原文，尤其不得大段复刻；
- 可以将摘录内容概括为主题、观点、关键词、倾向或写作素材方向；
- 如需引用原书内容，应遵守合理引用原则，并优先短引、注明来源，避免版权风险。

### 3.3 网络权威资料入口

使用目的：

- 核验事实、数据、定义、政策、标准、机构信息；
- 补充用户知识库中缺失的外部背景；
- 为观点提供公共可信支撑。

使用要求：

- 优先使用官方、原始、权威、近源资料；
- 搜索结果页只能作为发现入口，不可直接视为证据；
- 使用前应核对标题、发布机构、发布时间、正文上下文和适用范围；
- 对争议性、时效性强或高风险内容，应交叉验证多个来源；
- 不得为了支撑预设结论而选择性引用片面材料；
- 对无法确认真实性的资料，应明确标注“不确定”或降级为参考线索。

## 4. 关键判定规则

### 4.1 搜索命中不等于已验证

搜索命中仅表示系统找到了可能相关的入口，不代表：

- 内容真实；
- 内容最新；
- 来源权威；
- 观点适用于当前用户；
- 该资料已被用户阅读、认可或授权使用；
- 可以直接引用为事实证据。

正确处理方式：

1. 将搜索命中视为“候选资料”；
2. 打开或读取候选资料正文；
3. 核对来源、作者、发布时间、上下文；
4. 判断其与当前任务的相关性；
5. 必要时与其他资料交叉验证；
6. 再决定是否进入写作证据链。

### 4.2 授权阅读摘录优先级高于平台书目信息

在阅读平台、书目数据库或用户提供的阅读材料中，不同信息的优先级应区分如下：

1. 用户授权阅读摘录、个人笔记、个人评论；
2. 用户阅读状态、收藏状态、书架状态；
3. 书籍正文、目录、作者信息；
4. 平台简介、推荐语、标签；
5. 搜索结果标题或摘要。

原因：

- 授权阅读摘录更能反映用户实际关注点；
- 书目存在不代表用户认可；
- 平台推荐语通常是营销或摘要性质，不足以代表用户观点；
- 搜索摘要可能截断、误导或脱离上下文。

### 4.3 不得复制私人原文

涉及用户私人笔记、知识库、授权阅读摘录、会议记录、内部材料时，应遵守以下边界：

- 不得未经允许大段复制私人原文；
- 不得将私人内容包装成公开资料；
- 不得泄露姓名、联系方式、客户信息、项目细节、商业机密等敏感信息；
- 可以进行概括、脱敏、抽象、改写和结构化提炼；
- 如用户明确要求保留原文，应优先确认使用范围和交付对象。

## 5. 写作中的证据使用流程

建议流程如下：

1. 明确写作目标：判断任务是表达用户观点、生成对外文章、整理内部材料，还是做公开研究。
2. 先找用户个人证据：检查用户输入、历史偏好、授权知识库、项目材料、阅读摘录等。
3. 再找真实平台证据：核验书目、官方资料、权威来源、真实平台记录。
4. 最后补充公共方法论：用通用框架优化结构、节奏、标题、论证方式。
5. 区分证据状态：标记为“已验证事实”“用户个人观点”“平台候选资料”“公共方法论参考”。
6. 生成内容：优先体现用户个人判断，避免堆砌外部资料。
7. 输出前自检：确认没有把搜索命中当验证、没有复制私人原文、没有误用公共方法论冒充事实。

---

## 6. 推荐标注方式

在内部工作草稿中，可使用以下标注：

- `[L1-用户证据]`：来自用户输入、授权知识库、阅读摘录、项目材料、个人材料；
- `[L2-平台证据]`：来自官方网站、权威报告、公开数据库、出版方或书目平台等真实可核验来源；
- `[L3-方法论]`：来自通用写作、管理、传播、研究方法；
- `[候选-未验证]`：搜索命中但尚未打开或核验；
- `[需复核]`：存在时效、出处、适用范围或版权疑问；
- `[不可直引]`：私人原文、版权文本、敏感内部资料，仅可概括或脱敏使用。

对用户可见的最终交付中，不一定需要展示所有内部标签，但应确保引用和表述符合证据等级原则。

## 7. 风险与边界

高风险情形包括：

- 把搜索结果摘要当成事实；
- 把书目或平台命中当成用户读过或认可；
- 把公共方法论包装成用户个人经验；
- 复制用户私人笔记或划线原文；
- 引用过时政策、旧版标准或失效数据；
- 使用来源不明的二手材料；
- 为了增强说服力编造出处、作者、页码、数据或引用。

处理原则：

- 不确定就降级为参考线索；
- 不能核验就不要写成确定事实；
- 不能公开就只做概括和脱敏；
- 没有用户个人证据时，不要假装了解用户立场；
- 需要严肃引用时，必须优先真实核验。

---

## 8. 最小执行清单

每次调用外部资料前，至少检查：

- 是否先使用了用户个人证据？
- 搜索命中是否已经打开或核验？
- 书目命中是否区分了“书目存在”和“用户授权摘录”？
- 是否避免复制私人原文？
- 公共方法论是否只用于辅助，而非替代事实？
- 最终表述是否清楚区分“用户观点”“平台事实”“公共方法”？

---

## 9. 一句话原则

在「写作工作流」Skill 中，证据使用应始终遵循：

> 先尊重用户自己的证据，再核验真实平台资料，最后用公共方法论辅助表达；搜索命中不是验证，授权阅读摘录优先于书目信息，私人原文只能概括脱敏，不能直接复制。
